What & Why

Chetter is an MCP control plane that recruits AI agents as sub-contractors.

01General purpose

Any MCP-capable client (Claude, ChatGPT, OpenCode, curl) can talk to it. Not tied to one product.

02For AI, by AI

The control surface is tools, not a UI. An LLM is the typical operator.

03Remote, not local

Tasks run inside runner containers somewhere else. The MCP server holds no execution responsibilities.

04Durable queue

Tasks survive runner outages via JetStream. Schedules survive process restarts via MySQL.

05Observable fleet

Every task carries provider, model, variant, session id, and runner image digest for end-to-end traceability.

Task Lifecycle

From MCP tool call to runner event, in one linear flow.

1. SubmitInsertTask + random task_ id, then PublishTask to JS WorkQueue stream
2. QueueMessage sits on chetter.runner.tasks with WorkQueue policy; runners compete via queue group
3. RunRunner acks, spawns OpenCode agent, periodically publishes status to chetter.tasks.<id>.status
4. IngestChetter's event subscription decodes, persists raw payload, and updates the task row idempotently
5. CompleteTerminal state (done, error, cancelled) written once; replays ignored. Caller reads via task_status / task_events.

NATS JetStream

Two streams, two retention policies, two jobs.

CHETTER_TASKS

Durable task queue. WorkQueue retention — each task delivered to exactly one consumer.

subject: chetter.runner.tasks policy: work-queue delivery: one-to-one durable: chetter-runner queue group: chetter-runners

CHETTER_EVENTS

Replayable event log. Limits retention, manual ack, durable consumer per chetter instance.

subject: chetter.tasks.> policy: limits delivery: one-to-all durable: chetter-mcp-events ack: manual, 30s wait, 5 redeliveries

Adding a third runner is just a new container with the same durable name — Chetter never knows the fleet size.

Reaper & Health

No runner heartbeats — fleet health is inferred from task activity.

ReapStaleTasksrunning tasks older than timeout + 5 min → error
ReapStalePendingTaskspending tasks older than 30 min with no event → cancelled
GetRunnerFleetHealthjoins running tasks; is_stale = last_event_sec > 600
FleetActiverunning_tasks > stale_tasks (idle fleet looks dead)

Cheaper than heartbeats, but a fleet with no work reports zero active. Operators learn to read the absence-of-data as a sign of life.

Patterns & Tradeoffs

Choices that keep the codebase small and predictable.

Two-phase cancel

NATS cancel signal first, DB row update second. A failed publish is logged but the operator intent still persists.

ACK after write

Event subscription acks only after the DB write succeeds. Failed writes redeliver (up to 5 times). Idempotent updates keep replays safe.

Defensive env redaction

Keys matching SECRET|TOKEN|KEY|PASSWORD (case-insensitive) are scrubbed before both DB and wire paths. Users cannot smuggle secrets.

Timeouts on every call

NATS connect 10s, event handler 10s, schedule run 30s, Arcane 30s, HTTP read header 10s, shutdown 15s. No unbounded blocking.

Schema probes, not migrations

information_schema lookups at startup; ALTER TABLE if columns missing. Fine for add-only schema evolution on a small footprint.

One file per concern

service.go orchestration, tools.go MCP, store.go SQL, bus.go NATS, arcane.go HTTP. ~3k lines total. Easy to read end-to-end in a sitting.

Technical Choices

Each technology is biased toward operational simplicity and at-least-once delivery.

MCP, not REST

The control surface is for AI agents, not humans. Tools are the native interface for any LLM client.

Streamable HTTP

Modern MCP transport. Stateless mode makes the server trivially load-balancer-friendly.

NATS JetStream

Durable task queue + replayable event log in one broker. WorkQueue policy gives free load balancing across runners.

MySQL / TiDB

Same wire protocol as the rest of the stack. Small, add-only schema, conservative pool.

robfig/cron v3

Battle-tested, tiny, descriptor parser for @hourly-style. In-process is fine for single-instance.

Distroless binary

Multi-stage build → static-debian12:nonroot. No shell, no package manager, small surface.
